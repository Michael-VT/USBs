# Serial Terminal v05 - Продвинутый терминал для последовательных устройств

Serial Terminal v05 — это мощный терминал для работы с последовательными портами (USB/UART) с расширенными возможностями автоматического тестирования и управления устройствами.

## Основные возможности

### 🔧 Подключение и связь
- **Поддержка виртуальных и реальных портов**: VIRTUAL режим для тестирования без оборудования
- **Автоматическое переподключение**: При потере связи с устройством терминал автоматически пытается восстановить соединение
- **Настройки последовательного порта**: поддержка стандартных скоростей (9600, 115200 и др.)
- **HEX-отображение**: просмотр полученных данных в шестнадцатеричном формате

### 🚀 Автоматическое выполнение команд

#### Последовательное выполнение
- **Sequential Mode**: отправка команд из списка по порядку
- **Ожидание завершения**: следующая команда отправляется только после получения слова "Complete" в ответе
- **Настраиваемая задержка**: интервал между командами (0.2-1.0 секунды, по умолчанию 0.3 сек)
- **Таймауты**: 10-секундный таймаут после получения слова "Start"
- **Диапазонное тестирование**: запуск команд с N по M строку

#### Повтор команд
- **Repeat mode**: автоматическое повторение последней команды
- **Настройка интервала**: задание интервала между повторами (с защитой от зависания GUI)
- **Ограничение количества**: можно задать количество повторов или бесконечный режим

### 📊 Мониторинг и статистика

#### Command Statistics
- **Всего отправлено**: общее количество отправленных команд
- **Успешные выполнения**: команды, завершившиеся словом "Complete"
- **Таймауты**: команды, не завершившиеся в отведенное время
- **Среднее время ответа**: среднее время выполнения команд

#### Логирование
- **Автосохранение**: сохранение логов с автоматическим именем (дата-время-кол-во строк)
- **Копирование в буфер**: автоматически копирует содержимое лога в буфер обмена
- **Нормализация концов строк**: конвертирует \\r\\n и \\r в \\n для удобства чтения
- **Метаданные**: добавляет заголовок с датой, портом и количеством строк
- **Фильтрация пустых строк**: опционально пропускает пустые строки в логах

### 🎨 Настройки отображения

#### Визуализация
- **Темы оформления**: dark, amber, blue, white_on_black, white_on_blue
- **Номера строк**: опциональное отображение номеров строк
- **Временные метки**: опциональное добавление timestamps к каждой строке
- **Настройка шрифта**: изменяемый размер шрифта с горячими клавишами

#### Форматирование
- **EOL режим**: выбор концов строк при отправке данных
- **Фильтрация пустых строк**: включение/выключение фильтрации

### 🔍 Работа с командами

#### Список команд
- **40 команд**: список из 40 команд с возможностью редактирования
- **Нумерация**: каждая команда имеет номер для удобства навигации
- **Быстрая отправка**: одиночный клик по команде отправляет её
- **Редактирование**: двойной клик открывает диалог редактирования
- **Профили: сохранение и загрузка наборов команд

#### История команд
- **Автосохранение**: автоматически сохраняет отправленные команды
- **Персистенция**: история команд сохраняется в профиле

### 📁 Управление профилями

#### Сохранение настроек
- **Default profile**: автоматически сохраняет настройки по умолчанию
- **Именованные профили**: возможность создать именованные профили
- **Сохраняемые параметры**:
  - Список команд
  - Настройки порта и скорости
  - Тема оформления
  - EOL режим
  - Настройки фильтрации и отображения
  - Параметры выполнения (задержка, таймаут, паттерн)
  - История команд

### ⌨️ Горячие клавиши

#### Управление
- `Ctrl+X` — Выход из программы
- `Ctrl+B` — Очистка лога
- `Ctrl+C` — Копировать выделенный текст
- `Ctrl+S` — Сохранить лог (авто имя + копия в буфер)
- `Ctrl+P` — Сохранить профиль по умолчанию
- `Ctrl+F` — Поиск в логе
- `Ctrl+H` — Справка
- `Esc` — Вернуться к логу из справки
- `Ctrl+=` — Увеличить шрифт
- `Ctrl+-` — Уменьшить шриф

#### Автоматизация
- `Ctrl+Alt+M` — Переключение тема (день/ночь)
- `Ctrl+Alt+S` — Откройте настроек
- `Ctrl+Alt+R` — Начать выполнение последовательности
- `Ctrl+Alt+X` — Остановить выполнение

### 🎯 Тестирование и отладка

#### Поиск и анализ
- **Поиск в логах**: поиск текста с подсветкой совпадений
- **Экспорт фильтрованных логов**: сохранение только строк, содержащих заданный текст
- **HEX режим**: просмотр полученных данных в шестнадцатеричном виде

#### Виртуальный режим
- **VIRTUAL порт**: тестирование UI без подключения устройства
- **Эмуляция ответов**: отправка тестовых данных через Inject поле

## Использование

### Быстрый старт
1. Запустите программу: `python3 my05term.py`
2. Выберите порт (или VIRTUAL для тестирования)
3. Начните работу

### Последовательное тестирование
1. Заполните список команд в правой панели (двойной клик для редактирования)
2. В поле "Completion pattern" введите "Complete" (или ваше слово-маркер)
3. Настройте задержку (по умолчанию 0.3 сек) и таймаут (по умолчанию 10 сек)
4. Нажмите "▶ Start" для начала
5. Программа будет отправлять команды по порядку, ожидая "Complete"

### Повторение команд
1. Отправьте команду вручную через поле ввода
2. Нажмите кнопку "Start repeat" внизу
3. Команда будет повторяться с заданным интервалом
4. Нажмите "Stop repeat" для остановки

### Диапазонное тестирование
1. Укажите диапазон строк (например, с 1 по 10)
2. Убедитесь, что указан паттерн ожидания
3. Нажмите "▶ Run Range"
4. Программа выполнит команды только указанного диапазона

## Советы по использованию

### Производительность
- Используйте разумные значения задержек (0.3-0.5 сек для большинства устройств)
- Следите за таймаутами - увеличьте их для медленных устройств
- Используйте фильтрацию пустых строк для чистоты логов

### Отладка
- Используйте VIRTUAL порт для тестирования UI
- Проверяйте HEX отображение для анализа бинарных данных
- Используйте статистику для анализа производительности

### Оптимизация
- Настройте оптимальную задержку между командами
- Используйте диапазонное тестирование для проверки групп команд
- Сохраняйте профили для разных устройств/тестов

## Решение проблем

### Команды не отправляются
- Убедитесь, что указан правильный порт
- Проверьте, что установлен pyserial: `pip install pyserial`
- Проверьте наличие прав доступа к порту

### Таймауты при выполнении
- Увеличьте значение Timeout (s) в настройках
- Убедитесь, что устройство отправляет слово "Complete"
- Проверьте, что baud rate соответствует настройкам устройства

### UI зависает
- Уменьшите интервал повторения
- Проверьте, не установлена ли очень маленькая задержка
- Используйте кнопку "Stop" для прерывания выполнения
